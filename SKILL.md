# 创新创业大赛文档制作规范

> 本规范基于第八至第十二届互联网+创新创业大赛真实文件的深度分析，覆盖110个DOCX、50个XLSX、71个PDF和6个PPTX文件，提炼出可直接复用的制作标准。

---

## 一、Word文档（.docx）制作规范

### 1.1 页面设置

| 维度 | 通知/公文类 | 计划书/方案类 | 评审规则/聘书类 |
|------|-----------|-------------|----------------|
| 纸张 | A4纵向 21.0×29.7cm | A4纵向 21.0×29.7cm | A4横向 29.7×21.0cm |
| 上下边距 | 2.54cm | 2.54cm | 2.54cm |
| 左右边距 | 2.54cm | 2.00~2.54cm | 2.54cm |
| 页眉距 | 1.50cm | 1.50cm | 1.50cm |
| 页脚距 | 1.75cm | 1.75cm | 1.75cm |
| 装订线 | 0 | 0 | 0 |

### 1.2 字体字号规范

**通知/公文类文档（党政公文风格）：**

| 元素 | 中文字体 | 英文字体 | 字号 | 格式 |
|------|---------|---------|------|------|
| 大标题 | 方正小标宋简体 | Times New Roman | 20pt | 居中，不加粗 |
| 一级小标题 | 黑体 | Times New Roman | 16pt | 首行缩进2字符 |
| 二级小标题 | 楷体_GB2312 | Times New Roman | 16pt | 首行缩进2字符，加粗 |
| 正文 | 仿宋_GB2312 | Times New Roman | 16pt | 首行缩进2字符（1.13cm） |

**计划书/方案类文档（学术商业风格）：**

| 元素 | 样式名 | 字体 | 字号 | 行距 | 格式 |
|------|-------|------|------|------|------|
| 章标题 | Heading 1 | 宋体 | 18~22pt | 2.4倍 | 加粗，居中或左对齐 |
| 节标题 | Heading 2 | 宋体 | 14~16pt | 1.72倍 | 加粗 |
| 小节标题 | Heading 3 | 宋体 | 12~15pt | 1.72倍 | 加粗 |
| 正文 | Normal | 宋体 | 12~14pt | 1.5倍 | 首行缩进0.74~0.85cm |

**关键技巧：**
- 计划书必须使用Heading 1/2/3样式，配合自动编号（"第X章" → "X.X" → "X.X.X"）
- 通知类可不用Heading样式，直接在Normal段落中改变字体实现
- 标题颜色统一为黑色，不使用彩色
- 首行缩进统一用2字符，不要手动敲空格

### 1.3 表格规范

| 维度 | 标准 |
|------|------|
| 表格样式 | Table Grid（带可见边框） |
| 边框 | 单线，细线，黑色 |
| 表头 | 加粗，字号比正文大2pt |
| 表头底色 | 浅灰色 #CFCDCD（可选） |
| 对齐 | 居中对齐 |
| 表格宽度 | 百分比自适应或居中 |

### 1.4 图文排版技巧

- 计划书应适当配图（5~15张），图片居中，下方加图注说明
- 分栏仅在需要横向并排对比时使用
- 使用Word自带的List Paragraph样式做编号列表，不用手动编号
- 谨慎使用文本框（建议用表格替代），避免兼容性问题
- 分节符控制在2~3节以内，不要创建过多分节

### 1.5 页眉页脚

- 计划书/方案类：页眉写文档全称，页脚留空或加页码
- 通知/公文类：通常无页眉页脚
- 聘书/证书类：无页眉页脚
- 超过10页的文档应插入自动目录

### 1.6 避免的问题

- 页边距设为0或极小值
- 创建40+个分节符
- 仅依赖手动格式而忽略样式定义
- 首行缩进用空格代替标尺设置
- 使用过多不同字体（控制在3种以内）

## 二、Excel表格（.xlsx）制作规范

### 2.1 标准表格结构

**模板一：信息登记表（导师/人员名单）**

```
行1: 大标题（宋体24~48pt，加粗，居中，全列合并）
行2: 表头（宋体12~20pt，加粗，居中，thin边框）
行3: 示例行（红色字体，供填写参考）
行4+: 数据区（宋体11pt，居中，thin边框）
```

**模板二：汇总表（项目/参赛/获奖）**

```
行1: 标题（宋体14~24pt，加粗，居中，全列合并）
行2: 副标题/填表说明（宋体12pt，全列合并）
行3: 表头（宋体12pt，加粗，居中，thin边框）
行4+: 数据区（宋体11pt，thin边框）
```

**模板三：评分表**

```
行1: 大标题（华文中宋18pt，全列合并）
行2: 评分维度表头（宋体12pt，加粗，跨行/跨列合并）
行3+: 评分数据区
最后几行: 汇总/备注区（合并单元格）
```

### 2.2 字体规范

| 场景 | 中文字体 | 英文字体 | 字号 |
|------|---------|---------|------|
| 大标题 | 宋体 | Calibri | 18~48pt |
| 表头 | 宋体 | Calibri | 12~14pt |
| 数据行 | 宋体 | Calibri | 11pt |
| 公文标题 | 方正公文小标宋 | Times New Roman | 16~20pt |

全文统一使用宋体，英文用Calibri，严禁混用超过2种字体。

### 2.3 对齐与换行

| 维度 | 标准 |
|------|------|
| 水平对齐 | 全部居中（center） |
| 垂直对齐 | 全部居中（center） |
| 自动换行 | 数据区和表头必须启用（wrap_text=True） |

### 2.4 边框规范

- 统一使用 thin（细线）边框
- 颜色：黑色 #000000
- 四边完整：上、下、左、右均有边框

### 2.5 列宽参考

| 列内容类型 | 推荐宽度 |
|-----------|---------|
| 序号列 | 7~12 |
| 姓名列 | 10~16 |
| 学院/单位列 | 14~34 |
| 项目名称列 | 30~60 |
| 手机号/身份证列 | 15~26 |
| 项目简介/描述列 | 40~120 |
| 评分列 | 14~16 |

### 2.6 行高参考

- 标题行：30~70（通常41）
- 表头行：15~35（通常30）
- 数据行：15~20

### 2.7 合并单元格策略

- 标题行：必全列合并
- 副标题/填表信息行：全列合并
- 分类标题：跨列合并（如"队长基本信息"分组）
- 数据区底部备注：全列合并
- 数据区避免合并（影响排序和筛选）

### 2.8 数据验证与格式

- 固定选项列（赛道、组别、学院名称）设置下拉列表数据验证
- 手机号列设为文本格式（避免科学计数法）
- 超过一屏的表格应冻结标题行
- 列多（>10列）用横向打印，列少（≤8列）用纵向打印

### 2.9 避免的问题

- 字体混用超过2种
- 数据区合并单元格
- 不设边框或边框不完整
- 列宽未适配内容（文字被截断或过宽）
- 不冻结长表格的表头行
- 手机号/身份证号用数字格式

## 三、PDF文件制作规范

### 3.1 页面设置

- 标准尺寸：A4纵向（21.0×29.7cm）
- 横向使用：仅限证书、海报、评审表等特殊场景
- 页边距：上下左右各2.5cm

### 3.2 字体选择

| 元素 | 字体 | 字号 |
|------|------|------|
| 封面标题 | 微软雅黑 | 20~24pt |
| 大标题 | 微软雅黑或黑体 | 16~18pt |
| 章节标题 | 黑体 | 14pt |
| 正文（中文） | 宋体或等线体 | 11~12pt |
| 正文（英文） | Calibri或Times New Roman | 12pt |
| 页眉页脚 | 宋体或等线体 | 9~10pt |

关键技巧：
- 避免使用超过3种字体
- 字号变化不超过4个层级
- 全文保持一致
- 艺术字体仅用于封面或特殊强调

### 3.3 文档结构

标准项目计划书PDF结构：

1. 封面（1页）：大赛名称、项目标题、团队信息
2. 目录（1~2页）：自动生成，含页码链接
3. 执行摘要（1~2页）：项目亮点、创新点、市场机会
4. 正文章节（15~50页）：清晰章节标题，图文混排
5. 附录（5~10页）：详细数据、技术规格

- 页眉：大赛名称 + 项目主题
- 页脚：团队名称 + 页码
- 超过10页的文档必须添加书签/目录

### 3.4 排版技巧

- 行距：1.5倍
- 段前段后：6pt
- 首行缩进：2字符（0.75cm）
- 两端对齐
- 图片分辨率：300dpi以上
- 图片格式：照片用JPEG，图表用PNG
- 文字版PDF优于扫描版（可检索、可复制、文件更小）

### 3.5 PDF导出设置

- 版本：PDF 1.6或更高
- 压缩：启用ZIP或JPEG压缩
- 字体：嵌入所有使用的字体
- 兼容性：确保跨平台可读

### 3.6 避免的问题

- 扫描版PDF（无法检索文字）
- 图片分辨率不足（模糊）
- 未嵌入字体（换电脑显示异常）
- 文件过大（超过100MB）
- 超过10页无书签/目录

## 四、PPT演示文稿（.pptx）制作规范

### 4.1 基本规格

| 维度 | 标准 |
|------|------|
| 尺寸 | 16:9宽屏（13.3"×7.5"） |
| 页数 | 10~15页 |
| 路演时长 | 8~12分钟 |
| 每页展示时间 | 45~60秒 |
| 文件大小 | ≤50MB |

### 4.2 字体规范

| 元素 | 中文字体 | 英文字体 | 字号 |
|------|---------|---------|------|
| 主标题 | 微软雅黑 | Arial | 32~44pt |
| 副标题 | 微软雅黑 | Arial | 24~28pt |
| 正文 | 宋体 | Times New Roman | 16~20pt |
| 强调文字 | 黑体 | Helvetica | 16~20pt |
| 注释/脚注 | 宋体 | Times New Roman | 12~14pt |

全文不超过3种字体。

### 4.3 配色方案

3色原则：主色 + 辅色 + 强调色

| 方案名 | 主色 | 辅色 | 强调色 | 适用场景 |
|--------|------|------|--------|---------|
| 科技蓝 | #003366 | #FFFFFF | #66CCFF | 科技类项目 |
| 商务橙 | #333333 | #FFFFFF | #FF6600 | 商业类项目 |
| 生态绿 | #006633 | #F5F5F5 | #FFD700 | 农业/环保项目 |

### 4.4 版式结构

1. 封面页：Title Slide版式
2. 目录页：Table of Contents
3. 章节标题页：Section Header版式
4. 内容页：Title and Content版式
5. 对比页：Two Content / Comparison版式
6. 产品展示页：Picture with Caption版式
7. 结束页：Closing Slide

### 4.5 布局技巧

- 内容占比60~70%，留白30~40%
- 边距：上下左右各0.5~1英寸
- 单行文字不超过35个字符
- 行间距1.5~2倍
- 文本框左对齐为主，标题居中

### 4.6 图表与多媒体

- 图片分辨率≥300dpi，单张≤5MB
- 每张图表不超过6个数据系列
- 视频：H.264编码，MP4格式，时长<2分钟
- 动画：简洁为主，0.5~1秒，避免花哨效果
- 页面过渡：推荐无过渡或简单淡入

### 4.7 必备内容模块

1. 项目概述（1~2页）
2. 市场分析（2~3页）
3. 产品/技术介绍（2~3页）
4. 商业模式（1~2页）
5. 团队介绍（1页）
6. 财务预测（1~2页）
7. 融资计划（1页）
8. 联系方式（1页）

### 4.8 避免的问题

- 文字过多（每页超过6个要点）
- 使用过多花哨动画
- 图片模糊或拉伸变形
- 配色超过3种主色
- 字号过小（低于12pt）
- 超过15页（路演节奏拖沓）

## 五、通用规范

### 5.1 文件命名规范

```
[文档类型]-[项目名称]-[版本号].扩展名
```

示例：
- 项目计划书-智牧领航-v2.0.docx
- 项目汇总表-长春财经学院-2026.xlsx
- 路演PPT-棚联智检-决赛版.pptx

### 5.2 文件夹组织规范

```
项目名称/
├── 01-项目计划书/
├── 02-路演PPT/
├── 03-支撑材料/
├── 04-通知与规范/
└── 05-证书与聘书/
```

### 5.3 通用排版检查清单

**Word文档检查项：**
- 页面设置（A4、页边距2.54cm）
- 标题使用Heading样式
- 正文宋体12~14pt，1.5倍行距
- 首行缩进2字符
- 表格使用Table Grid样式
- 超过10页插入自动目录
- 图片居中，下方有图注

**Excel表格检查项：**
- 标题行全列合并，宋体18pt+加粗
- 表头宋体12pt加粗，居中
- 数据行宋体11pt，居中，thin边框
- 自动换行已启用
- 长文本列已适配列宽
- 固定选项列设数据验证
- 长表格冻结表头行

**PDF文件检查项：**
- A4纵向，页边距2.5cm
- 字体已嵌入
- 图片≥300dpi
- 超过10页有书签
- 文字可检索（非扫描版）

**PPT演示检查项：**
- 16:9宽屏
- 10~15页
- 不超过3种字体
- 不超过3种主色
- 每页不超过6个要点
- 图片清晰（≥300dpi）
- 动画简洁（0.5~1秒）
